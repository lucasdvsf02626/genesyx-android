## What shipped

### 1. Weekly Summary card (Insights)
- `ui/insights/WeeklySummaryLogic.kt` (+ `WeeklySummaryInsights` in `InsightsViewModel`) — current
  Mon–Sun week vs the one before: days logged, mood/energy tally, hydration/sleep/supplement deltas
  shown **only when both weeks hold data**. Card placed after *My logs*, before *Consistency*.
- Extracted the single "meaningful log" predicate from `StreakEngine` into shared
  `DailyLog.isMeaningful()` so summary + streak count days identically. `StreakEngineTest` +
  `TrackingVectorTest` unchanged → cross-platform contract intact.
- Tests: `WeeklySummaryLogicTest` (9).

### 2. Local reminders (`notifications/`)
- Strictly on-device (WorkManager, no FCM/token). `FeatureFlags.PUSH_NOTIFICATIONS = true`.
- Model: `ReminderKind` (6), `NotificationSettings`. Engine: `ReminderPolicy` (pure — quiet-hours
  overnight wrap, already-logged-today, daily cap, re-engagement pacing/hard-stop) +
  `NotificationPermission` (API-33 "dialog shows twice" trap). IO: `NotificationChannels` (4),
  `ReminderScheduler`/`WorkManagerReminderScheduler` (self-rescheduling one-time chain),
  `ReminderWorker` (Hilt EntryPoint), `ReminderNotifier`, `ReminderContent`, `AppForegroundTracker`,
  `BootReceiver`. Store: `data/NotificationSettingsRepository` (reuses `push_enabled` as master).
- UI: `ui/settings/ReminderSettingsScreen` + `ViewModel` + `PrePermissionSheet`; Profile → Reminders
  nav row; `POST_NOTIFICATIONS` handling incl. open-settings routing.
- Integration: channels created in `GenesyxApplication`; `MainActivity` singleTop + `onNewIntent` +
  foreground/opened tracking; manifest permissions + receiver; nav deep links; `cancelAll()` on
  sign-out/delete (`AuthRepository`).
- Tests: `ReminderPolicyTest` (19), `NotificationPermissionTest` (8), `ReminderNotifierInstrumentedTest`
  (on-device — a reminder fires and posts through `tracking_reminders`).

### 3. Intraday hydration coaching
- `domain/hydration/HydrationCoach.kt` — compares intake vs how much of the waking day has passed,
  framed morning/afternoon/evening, never a grade. Wired into Home + Nutrition hydration cards.
- Tests: `HydrationCoachTest` (8).

### 4. Home / Track / Insights iOS parity
- **Track "Your Trackers"** (`ui/track/TrackScreen`, `TrackerSummaryLogic`) — 6 rows with real
  summaries + spark dots + a11y labels; embedded pH moved to its detail.
- **Six tracker detail screens** (`ui/track/detail/`): `TrackerDetailScaffold`, `HydrationDetail*`
  (canonical editor: quick-add + manual entry clamped, 7-day bars, days-on-goal, streak, daily
  history — writes only via `DailyLogRepository`), `PhDetailScreen` (wraps `PhTrackerSection`),
  `CycleDetailScreen`, `SleepDetailScreen`, `SymptomsDetailScreen`, `NutritionDetailScreen`.
- **Home cards** (`ui/home/HomeScreen`, `HomeViewModel`): hydration-ring card (60dp ring, status
  pill, streak flame, coaching, 7 day-dots, "X of 7 on goal") + pH-nudge card, both deep-linking to
  the Track detail (select Track → push detail); first-run "Welcome to Genesyx" card
  (`PastDatePickerDialog`); cycle-hero 3 metrics; 44dp gradient avatar; **pregnancy entry hidden**.
- Shared components: `HydrationStatusPill`, `HydrationGoalDialog`.
- **Insights**: "Supplements" → "Nutrition consistency"; the 10 canonical cards in spec order + the
  Weekly Summary extra.
- **Nav**: 6 typed detail routes in `Screen`/`GenesyxNavGraph`; deep links `genesyx://tracker/{hydration,ph}`.
- Tests: `TrackerSummaryLogicTest` (7); `TrackViewModelTest` updated for the new dependency.
- **Decisions**: Weekly Summary kept as a documented extra; pregnancy hidden (route/feature kept);
  pH detail reuses the existing visualization (no new chart); single source of truth (no screen-owned
  totals).
